Toggle Component Design System DocumentationOverview
The toggle component is a UI element that allows users to switch between two states: on and off. It's a widely used component in modern web and mobile applications, and it can be used for various purposes such as turning features on or off, activating/deactivating settings, and more.
AnatomyThe toggle component consists of the following elements:
Toggle switch: This is the main element that allows the user to switch between the on and off states. It's typically represented as a horizontal bar with a circular or rectangular knob that can be dragged left or right.
Label: This is the text that appears next to the toggle switch to provide context about what the switch is for.

TypesThe toggle component comes in one type, which is medium-sized (48px).StatesThe toggle component has the following interaction states:
Default: This is the normal state of the toggle, with no interaction or modification applied.
Hover: This state is activated when the user hovers over the toggle switch. It provides visual feedback to indicate that the switch is interactive.
Pressed: This state is activated when the user clicks and holds the toggle switch. It provides visual feedback to indicate that the switch is being manipulated.
Disabled: This state is activated when the toggle is not available for interaction. It provides visual feedback to indicate that the switch is inactive and cannot be toggled.
Focused: This state is activated when the user navigates to the toggle using the keyboard. It provides visual feedback to indicate that the toggle is the currently active element.
Error: This state is activated when there is an error with the toggle, such as a validation error or an invalid input. It provides visual feedback to indicate that there is an issue with the toggle.
Read-only: This state is activated when the toggle is read-only and cannot be modified. It provides visual feedback to indicate that the switch is inactive and cannot be toggled.
Best PracticesDo
Use clear and concise labeling that accurately describes the toggle's functionality.Make sure the toggle's states are clearly defined and easy to understand.Ensure that the toggle provides visual feedback when it is interacted with or its state changes.Use consistent colors and styles for the toggle across the entire application.
Don't
Use ambiguous labeling that can confuse users about the toggle's functionality.Use inconsistent colors or styles for the toggle across the application.Use complex or non-standard toggle designs that may confuse or frustrate users.Use the toggle for tasks that require more precise or complex input, such as selecting from a range of values.
Accessibility InformationThe toggle component should meet the following accessibility requirements:
Keyboard accessibility: The toggle should be operable using the keyboard alone, without requiring the use of a mouse or other pointing device.
Screen reader accessibility: The toggle should be announced to screen reader users, with appropriate labeling and descriptions.
Color contrast: The toggle should have sufficient color contrast between its various states to be easily distinguishable by users with low vision.
Focus indicators: The toggle should provide clear and consistent focus indicators to indicate the currently active element for users who navigate with the keyboard.
Conclusion
The toggle component is a simple yet powerful UI element that can be used to enhance the user experience in various applications. By following the best practices outlined above and ensuring that the component meets accessibility requirements, you can create a toggle that is both functional and user-friendly.
